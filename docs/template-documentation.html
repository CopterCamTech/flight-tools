Flight-Tools HTML Template Structure

This document outlines the standard structure used in HTML templates for Flask-based tools in the Flight-Tools project. These templates are standalone files (not Jinja2 extensions) and follow a consistent pattern for clarity, maintainability, and contributor onboarding.

Standard Template Layout

Most tools follow this structure:

  1. DOCTYPE and HTML Head
    • Includes <!DOCTYPE html>, <html>, and <head> with a descriptive <title>.
  2. Body Header
    • Tool-specific heading (e.g., <h1>ArduPilot Parameter List</h1>).
  3. Conditional Upload Form
    • Displayed only when no results are present.
    • Uses method="post" and enctype="multipart/form-data".
    • Field name matches Flask route logic (e.g., logfile or file).
  4. Result Block
    • Conditionally rendered using {% if summary %}.
    • Displays filename, error messages, or extracted data (usually in a table).
    • Fallback messaging for empty or invalid logs.
  5. Navigation Footer
    • Always includes a link to restart the workflow (e.g., Upload another file).

Example Snippet

<form method="post" enctype="multipart/form-data">
    <input type="file" name="logfile" accept=".BIN">
    <input type="submit" value="Upload">
</form>

{% if summary %}
    <table>...</table>
{% endif %}

<a href="/bin-parameter-list">Upload another file</a>

  • Future Tools
    • May include search boxes, filtering, or download links.
    • Should follow the same fallback and conditional rendering principles.

Best Practices

  • Use semantic HTML for clarity and accessibility.
  • Match form field names to Flask route logic.
  • Always include fallback messaging for errors or empty results.
  • Keep templates modular and visually consistent across tools.
